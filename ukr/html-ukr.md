## Що таке _«HTML»_?
__HTML__, HyperText Markup Language («мова гіпертекстової розмітки») — стандартизована мова розмітки документів у WWW. Це не мова програмування, а мова розмітки: вона описує *структуру й семантику* вмісту (заголовки, абзаци, посилання, форми), а не алгоритми.

Актуальна редакція — HTML5, яку W3C і WHATWG розвивають як «живий стандарт» (Living Standard): версії на кшталт «HTML 5.1/5.2» більше не нумеруються, специфікація просто еволюціонує.

## Що таке _«XHTML»_?
__XHTML__, eXtensible HyperText Markup Language («розширювана мова гіпертекстової розмітки») — суворіший варіант HTML, що дотримується всіх обмежень XML і фактично є застосуванням мови XML до області розмітки гіпертексту.

На практиці XHTML сьогодні майже не використовують: екосистема перейшла на HTML5, який поєднує зручність HTML із можливістю суворого XML-серіалізатора (XHTML5) там, де це справді потрібно (наприклад, обробка документа XML-інструментами). Суворість XML мала ціну: одна синтаксична помилка (незакритий тег, неекранований `&`) призводила до повної відмови рендерингу сторінки, тоді як HTML-парсер пробачає такі дрібниці.

## Що таке `DOCTYPE` і навіщо він потрібен?
Елемент `<!DOCTYPE>` оголошує тип поточного документа. Він потрібен, щоб браузер розумів, за яким стандартом інтерпретувати цю web-сторінку.

Ментальна модель: `DOCTYPE` — це перемикач режиму рендерингу. Якщо його немає (або він некоректний), браузер вмикає **quirks mode** — режим сумісності зі старими сторінками 1990-х, де модель блоків і інші правила поводяться «неправильно». Коректний `DOCTYPE` вмикає **standards mode** — сучасну, передбачувану поведінку. Саме тому в HTML5 декларація гранично коротка і єдина, що потрібно пам'ятати на практиці:

__HTML5__

+ `<!DOCTYPE html>`: для всіх документів.

Історичні (застарілі) форми `DOCTYPE` для HTML 4.01 і XHTML 1.0/1.1 містили посилання на DTD і виглядали громіздко, наприклад:

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
"http://www.w3.org/TR/html4/strict.dtd">
```

Сьогодні ці варіанти на нових проєктах не використовують — їх варто знати лише для розуміння легасі-коду. У HTML5 DTD скасовано, тому `DOCTYPE` зведено до простого перемикача режиму.

## Для чого призначений тег `<head>`?
Тег `<head>` зберігає метадані документа — інформацію *про* сторінку, а не сам видимий вміст. Сюди потрапляє те, що потрібно браузеру, пошуковим системам і соцмережам для коректної обробки сторінки.

Вміст `<head>` не відображається безпосередньо на сторінці, за винятком тега `<title>`, який задає заголовок вкладки/вікна.

Усередині `<head>` зазвичай розміщують такі елементи: `<title>`, `<meta>` (кодування, viewport, опис, ключові слова, Open Graph тощо), `<link>` (підключення CSS, іконки, canonical), `<style>`, `<script>`, `<base>`.

> Примітка: теги `<basefont>` і `<bgsound>` із старого HTML більше не входять до стандарту і не повинні використовуватися; атрибут `profile` тега `<head>` також прибрано в HTML5.

Синтаксис:

```html
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Назва сторінки</title>
</head>
```

## Чим відрізняється `<div>` від `<span>`?
`<div>` — блоковий елемент, а `<span>` — рядковий (інлайновий). Тому `<div>` починається з нового рядка й займає всю доступну ширину, а `<span>` залишається в потоці тексту, не переносячи рядок.

| Критерій | `<div>` | `<span>` |
|---|---|---|
| Тип | блоковий | рядковий |
| Перенесення рядка | так, з нового рядка | ні, у тому ж рядку |
| Типове застосування | розмітка блоків/секцій | виділення фрагмента тексту |
| Може містити блокові елементи | так | ні (лише рядкові) |

За специфікацією рядковий елемент не може містити блокові, тому `<div>` зазвичай використовують для розмітки контейнерів, а `<span>` — для стилізації окремих шматків тексту всередині рядка.

> На практиці: обидва теги семантично «порожні». Якщо вміст має зміст (навігація, стаття, бічна панель, футер), краще брати семантичні теги HTML5 — `<nav>`, `<article>`, `<section>`, `<aside>`, `<footer>` — а `<div>`/`<span>` лишати для випадків, коли семантики справді немає.

## Як позначаються коментарі в HTML?
Коментар в HTML задається так: `<!-- коментар -->`

Коментарі можна розміщувати майже будь-де, але вони не працюють усередині тексту тега `<title>`. Усередині `<style>` і `<script>` HTML-коментарі теж не діють, бо в CSS і JavaScript власний синтаксис коментарів (`/* ... */` і `//`).

> Важливо: HTML-коментарі видно в початковому коді сторінки (View Source), тому не лишайте в них чутливих даних — паролів, внутрішніх URL, токенів.

## Яким чином задається адреса документа, на який слід перейти?
Для створення посилань використовується тег `<a>`. Адресу переходу задає атрибут `href`.

Синтаксис:

```html
<a href="URL">текст посилання</a>
```

Найуживаніші атрибути:

+ `href`: задає адресу документа, на який слід перейти. Адреса може бути абсолютною (`https://example.com/page`) або відносною (`../page.html`, `/path`). Якщо `href` указує на якір (`#section`), відбувається перехід до елемента з відповідним `id` усередині сторінки;
+ `target`: ім'я вкладки/вікна, куди завантажиться документ (наприклад, `_blank` — нова вкладка);
+ `rel`: відношення між поточним і цільовим документами. Для зовнішніх посилань із `target="_blank"` важливо додавати `rel="noopener noreferrer"` із міркувань безпеки (інакше нова вкладка отримує доступ до `window.opener`);
+ `download`: пропонує завантажити файл за посиланням замість відкриття;
+ `hreflang`: ідентифікує мову тексту за посиланням;
+ `type`: указує MIME-тип цільового документа;
+ `title`: додає спливну підказку до посилання;
+ `accesskey`: активація посилання комбінацією клавіш;
+ `tabindex`: визначає послідовність переходу між елементами клавішею <kbd>Tab</kbd>.

> Примітка щодо легасі: у старому HTML якір створювали через `<a name="...">`, а атрибути `coords`, `shape`, `rev` стосувалися застарілих карт-зображень. У сучасному HTML5 для якоря використовують атрибут `id` на будь-якому елементі (`<h2 id="section">`), а `name` для цього вважається застарілим.

## Як зробити посилання на адресу електронної пошти?
Робиться майже так само, як звичайне посилання, тільки замість URL у `href` указують схему `mailto:` з адресою:

```html
<a href="mailto:user@address.net">Напиши мені!</a>
```

Можна одразу додати тему й текст листа через параметри:

```html
<a href="mailto:user@address.net?subject=Привіт&body=Текст%20листа">Напиши мені!</a>
```

Перехід за таким посиланням відкриє поштовий клієнт користувача за замовчуванням.

## Для чого призначений тег `<em>`?
Тег `<em>` (emphasis) позначає *смисловий акцент* — слово чи фразу, яку при читанні слід виділити інтонацією. Браузери за замовчуванням відображають такий текст курсивом.

```html
<p>Це <em>дуже</em> важливо.</p>
```

> Не плутати з `<i>`: `<em>` несе семантику наголосу (важливо для скрінрідерів і SEO), а `<i>` — суто візуальний курсив без зміни змісту (терміни, назви, іншомовні слова). Так само `<strong>` означає важливість, а `<b>` — лише жирне накреслення без семантики.

[до змісту](#Основи-html)

## Для чого призначені теги `<ol>`, `<ul>`, `<li>`?
Ці теги призначені для оформлення списків.

+ `<ol>` (ordered list): нумерований список — порядок елементів має значення (кроки інструкції, рейтинг);
+ `<ul>` (unordered list): маркований список — порядок неважливий (перелік можливостей);
+ `<li>` (list item): окремий елемент списку. Зовнішній тег `<ul>` або `<ol>` визначає тип списку.

```html
<ol>
    <li>перший</li>
    <li>другий</li>
    <li>третій</li>
</ol>

<ul>
    <li>перший</li>
    <li>другий</li>
    <li>третій</li>
</ul>
```

> Виправлено: в оригіналі текст («Нумерований список») був написаний прямо всередині `<ol>`/`<ul>` поряд з `<li>`. Це некоректна розмітка — безпосередніми дітьми `<ol>`/`<ul>` мають бути лише `<li>` (а також `<script>`/`<template>`). Текстовий підпис слід виносити за межі списку, наприклад у заголовок чи абзац перед ним.

## Для чого призначені теги `<dl>`, `<dt>`, `<dd>`?
Теги `<dl>`, `<dt>`, `<dd>` призначені для створення списку визначень (description list) — наборів пар «термін → опис»: глосарій, перелік метаданих, FAQ.

Список починається з контейнера `<dl>`, куди входять тег `<dt>` (термін) і тег `<dd>` (визначення цього терміна). Закривальний тег `</dd>` формально не обов'язковий, бо наступний тег сигналізує про завершення попереднього елемента, проте закривати всі теги — хороший стиль.

```html
<dl>
    <dt>Термін</dt>
    <dd>Визначення</dd>
</dl>
```

> Виправлено: як і вище, текст «Список визначень» в оригіналі стояв прямо в `<dl>` поза парами `<dt>`/`<dd>` — це невалідно й прибрано.

## Для чого призначені теги `<tr>`, `<th>`, `<td>`?

+ `<tr>` (table row): контейнер для рядка таблиці. Кожна комірка в межах рядка задається тегом `<th>` або `<td>`.
+ `<th>` (table header): комірка-заголовок таблиці (за замовчуванням текст напівжирний і вирівняний по центру).
+ `<td>` (table data): звичайна комірка з даними.

```html
<table>
    <tr>
        <th>Заголовок</th>
    </tr>
    <tr>
        <td>Рядок</td>
    </tr>
</table>
```

> Виправлено: в оригіналі комірку заголовка було відкрито як `<th>`, але закрито як `</td>` — теги не збігалися. Правильно: `<th>Заголовок</th>`.

> Для семантики й доступності великі таблиці варто розбивати на секції `<thead>`, `<tbody>`, `<tfoot>`, а заголовкам додавати атрибут `scope="col"`/`scope="row"`.

## Чи обов'язково писати атрибут `alt` у тезі `<img>`?
Так, атрибут `alt` обов'язковий за специфікацією HTML.

`alt` задає альтернативний текст зображення. Він потрібен у трьох ситуаціях:
- зображення не завантажилося (битий шлях, повільна мережа) — користувач побачить текст замість картинки;
- екранні читачі (для людей з вадами зору) озвучують саме `alt` — без нього зображення для них «невидиме»;
- пошукові системи індексують `alt` як опис картинки.

```html
<img src="forest.jpg" alt="Ліс">
```

> Виняток: якщо зображення суто декоративне й не несе інформації, його лишають з порожнім `alt=""` — тоді скрінрідер його пропускає. Тобто атрибут присутній завжди, але для декору він порожній.

## У якому регістрі краще писати HTML-код?
Хоча HTML нечутливий до регістру тегів і атрибутів, за усталеним стилем увесь код пишуть у нижньому регістрі: назви елементів, назви й значення атрибутів (окрім тексту/`CDATA`), CSS-селектори, властивості та їхні значення (окрім тексту). Це підвищує читабельність і узгоджується зі стилем-гайдами (зокрема Google HTML/CSS Style Guide).

Не рекомендується:
```html
<A HREF="/">Додому</A>
```

Рекомендується:
```html
<a href="/">Додому</a>
```

> Виправлено: у «рекомендованому» прикладі оригіналу замість виправленого посилання було помилково продубльовано тег `<img>` з попереднього питання. Тут наведено той самий приклад посилання, але в коректному нижньому регістрі.

## Що таке «мнемоніка (entity)»?
__Мнемоніка (entity, символьне посилання)__ — це конструкція із символу `&`, коду і крапки з комою (`;`), яка заміщує символи, що не можна (або незручно) писати в HTML «у явному вигляді». Зокрема, спеціальні символи самої розмітки: `&lt;` (`<`), `&gt;` (`>`), `&amp;` (`&`), `&quot;` (`"`).

Мнемоніки бувають іменовані (`&num;`) і числові — у десятковому (`&#35;`) чи шістнадцятковому (`&#x23;`) вигляді.

> Символ `#` має мнемоніку `&num;`
