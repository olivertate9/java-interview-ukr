## Що таке _«CSS»_?
__CSS, Cascading Style Sheets (каскадні таблиці стилів)__ — формальна мова опису зовнішнього вигляду документа, написаного за допомогою мови розмітки. Вона застосовується до елементів web-сторінки, щоб керувати їхнім виглядом і розташуванням.

Головна ідея CSS — відокремити опис логічної структури web-сторінки (її роблять засобами HTML або іншої мови розмітки) від опису зовнішнього вигляду цієї сторінки (його роблять засобами CSS). Це дає змогу змінювати оформлення, не чіпаючи розмітку, і повторно використовувати одні й ті самі стилі на багатьох сторінках.

## Як у CSS позначаються коментарі?
Щоб позначити фрагмент як коментар, застосовують конструкцію `/* ... */`. Однорядкових коментарів (на кшталт `//`) у CSS немає — будь-який коментар обрамлюється `/* */`.

## Що таке _«селектор»_?
__Селектор__ — це правило, на підставі якого здійснюється вибір елементів у HTML-документі, щоб застосувати до них певні стилі.

```css
p {
    text-align: center;
    font-size: 20px;
}
/* p — це селектор, text-align і font-size — це властивості, а center і 20px — значення. */
```

[до змісту](#Основи-css)

## Перелічіть основні види селекторів.

+ __селектор `*`__ — вибір усіх елементів;
+ __селектор елемента (типу)__ — вибір усіх елементів у HTML-документі, що мають указаний тег (наприклад: `div`);
+ __селектор класу__ — вибір усіх елементів, що мають указаний клас (наприклад: `.center`);
+ __селектор ідентифікатора__ — вибір елемента, що має вказаний ідентифікатор (наприклад: `#footer`);
+ __селектори псевдокласів__ — вибір елементів, що перебувають у певному стані (наприклад: `p:first-of-type`);
+ __селектори атрибутів__ — вибір елементів залежно від указаного атрибута або його значення (наприклад: `[href*="youtube"]`).

Окрім них, на практиці постійно застосовують __комбінатори__, що поєднують прості селектори за взаємним розташуванням елементів: нащадок (`div p`), прямий нащадок (`div > p`), сусідній (`h1 + p`) та наступні елементи того ж рівня (`h1 ~ p`).

## Що таке псевдоклас?
Псевдоклас визначає динамічний стан елемента, який змінюється через дії користувача, або ж відповідає поточному положенню елемента в дереві документа. На відміну від звичайного класу, псевдоклас явно в HTML не вказується, а в CSS дописується через `:` безпосередньо після селектора.

Найвідоміші псевдокласи:

+ `:link` застосовується до невідвіданих посилань;
+ `:visited` застосовується до відвіданих посилань;
+ `:hover` застосовується, коли курсор миші перебуває в межах елемента;
+ `:active` застосовується в момент активації елемента (наприклад, поки натиснута кнопка миші);
+ `:focus` застосовується до елемента, коли він отримує фокус;
+ `:first-child` застосовується до елемента, який є першим дочірнім у свого батька;
+ `:nth-child(n)` застосовується до елемента, що є n-м дочірнім (наприклад, `:nth-child(2n)` — кожен парний).

```css
a.snowman:link {
    color: blue;
}
a.snowman:visited {
    color: purple;
}
a.snowman:active {
    color: red;
}
a.snowman:hover {
    text-decoration: none;
    color: blue;
    background-color: yellow;
}
```

Порада щодо порядку: щоб посилання поводилися передбачувано, псевдокласи зазвичай оголошують у порядку LVHA — `:link`, `:visited`, `:hover`, `:active`. Інакше пізніше правило з однаковою специфічністю може перекрити попереднє.

## Які існують селектори атрибутів?
+ __`[атрибут]`__ — усі елементи, що мають указаний `атрибут`;
+ __`[атрибут=значення]`__ — усі елементи, у яких `атрибут` точно дорівнює `значення`;
+ __`[атрибут^=значення]`__ — усі елементи, у яких значення атрибута починається з `значення`;
+ __`[атрибут|=значення]`__ — усі елементи, у яких значення атрибута дорівнює `значення` або починається з `значення-` (тобто `значення` з обов'язковим дефісом, після якого йде решта вмісту); зазвичай застосовують для мовних кодів на кшталт `en-US`;
+ __`[атрибут$=значення]`__ — усі елементи, у яких значення атрибута закінчується на `значення`;
+ __`[атрибут*=значення]`__ — усі елементи, у яких значення атрибута містить підрядок `значення`;
+ __`[атрибут~=значення]`__ — усі елементи, у яких значення атрибута містить `значення` як одне з кількох значень, розділених пробілами.

## Яка різниця між `#my` і `.my`?
`#my` — селектор ідентифікатора (вибирає елемент з `id="my"`), а `.my` — селектор класу (вибирає всі елементи з `class="my"`).

Окрім сфери дії, важлива й __специфічність__: ідентифікатор «важчий» за клас, тому за конфлікту правил перемагає стиль, заданий через `#my`.

[до змісту](#Основи-css)

## Яка різниця між `margin` і `padding`?
`margin` — це зовнішній відступ (простір за межами рамки елемента, між ним і сусідами), а `padding` — внутрішній відступ (простір між вмістом елемента та його рамкою `border`).

Ще одна практична відмінність: вертикальні `margin` сусідніх блоків можуть __схлопуватися__ (margin collapsing) — береться більший з двох, а не їхня сума; з `padding` такого не відбувається. Крім того, фон елемента (`background`) поширюється на область `padding`, але не на `margin`.

[до змісту](#Основи-css)

## У чому полягає різниця між значеннями `0` і `auto` у властивості `margin`?
`0` явно прибирає відступ. `auto` доручає браузеру обчислити відступ автоматично, і поводиться по-різному залежно від напрямку:

+ __горизонтальні__ відступи (`margin-left`/`margin-right`): за наявності заданої `width` блокового елемента значення `auto` рівномірно розподіляє вільний горизонтальний простір — саме так блок центрують по горизонталі (`margin: 0 auto`). Якщо `width` не задана, `auto` дорівнює `0`;
+ __вертикальні__ відступи (`margin-top`/`margin-bottom`): для звичайних блоків `auto` дорівнює `0` — вертикального центрування таким способом досягти не можна. (У сучасних розкладках flexbox і grid `margin: auto` працює й по вертикалі та дозволяє центрувати елемент в обох напрямках.)

[до змісту](#Основи-css)

## Яка властивість задає колір тла?
Колір тла задає властивість `background-color`. Її також можна задати скорочено через `background` (наприклад, `background: #fff`), яка водночас керує зображенням, позицією та повторенням тла.

## Для чого використовується властивість `clear`?
`clear` визначає, з якого боку елемента заборонено обтікання плаваючими (`float`) елементами: елемент із `clear` зсувається вниз, щоб опинитися нижче відповідних float. Можливі значення: `left`, `right`, `both`, `none`.

Класичне застосування — «розкрити» контейнер, висоту якого «з'їли» плаваючі нащадки (clearfix). Втім, у сучасній верстці для таких розкладок переважно беруть flexbox або grid замість `float` і `clear`.
